The container comes with this simple test site automatically. Clearly you have worked out how to start the container. You may however be wondering how you can manage and start testing your own Django web apps.

Importing Your Own Django App

There are numerous methods of adding your content into the container but first it is important to understand the expected layout.

  • /var/www - This is where the root of your app should be.
  • /var/www/static - This folder should contain static files such as images and css files.
  • /var/www/media - This folder should contain media files such as videos.

The following command will map the specified paths to those mentioned above. The advantage of mapping volumes in this manner is that you can edit files and they will reflect immediately

            docker run -itd -v c:/path/to/app /var/www
                            -v c:/path/to/static /var/www/static
                            -v c:/path/to/media /var/www/media
                            -p 8080:80
                            django petermcd/django-uwsgi-nginx

After mounting the volumes you will need to carry out 1 further modification. You will be required to edit start.sh to point wsgi to point to the correct project.
